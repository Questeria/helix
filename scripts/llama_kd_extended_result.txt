# === JUDGMENT (honest) ===
# MORE EPOCHS HELPED -- to a point. Extending KD from 5 to 13 epochs pushed the best held-out ternary
# ppl from 1278.95 (the 5-epoch endpoint) DOWN to 1140.55 at epoch 10 (10.8% better), the lowest
# converted-ternary ppl on this corpus to date (130.4x the fp 8.7467; ~half hard-CE's 2302). So the
# 5-epoch "still falling" read was correct -- the model HAD more to give past epoch 5.
#
# BUT the trajectory is NON-MONOTONE and NOISY, and it has now turned over. The held-out ppl wobbled
# (e7 1205 -> e8 1231 -> e9 1464 -> e10 1140) and after the epoch-10 minimum it rose steadily
# (e11 1223, e12 1302, e13 1900) while the train mean-loss kept low/falling -- the textbook OVERFIT
# signature on a tiny corpus. The wobble itself is a measurement-noise artifact of the SMALL held-out
# set (only 1166 predicted tokens, 5 windows): single-window swings move the geometric-mean ppl by
# tens-to-hundreds, so individual epoch numbers past ~e6 are within the noise band.
#
# VERDICT on "more epochs vs bigger corpus": CORPUS SIZE is now the binding limit, NOT epoch count.
# Evidence: (a) the model overfits 4303 train tokens by ~epoch 10-11 (held-out rises, train-loss low);
# (b) the held-out estimator is too small to resolve sub-percent epoch-to-epoch gains -- the signal is
# in the noise. Running yet more epochs on THIS corpus would only deepen the overfit. The next lever is
# a LARGER, more diverse train corpus (and a larger held-out set to measure it), with early-stopping on
# held-out (exactly what HX_SAVE.best now captures automatically). The gap to fp (130x) remains large
# and consistent with the standing finding that a real ternary conversion needs real-scale data + a
# longer schedule on THAT data -- KD is a genuine, measurable lever (best ppl ~half of hard-CE), but it
# did not and cannot close the gap on a 4303-token set.
#
# SHIP: the epoch-10 checkpoint (HX_SAVE.best, 1140.55), NOT the final epoch-13 weights (1900.39, overfit).
