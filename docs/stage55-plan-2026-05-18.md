## Increment breakdown (7 incs)

### Inc 1 â€” Runtime string builtins (LOW, ~200 LOC Python)

Add: `__str_len(start, len)`, `__str_byte_at(start, pos)`,
`__str_find_byte(start, len, byte_val)`, `__str_slice(start,
off, n)`. All operate on `(start: i32, len: i32)` arena-backed
strings. Lower to existing ARENA_GET + arithmetic IR ops.

Files: `typecheck.py` whitelist, `lower_ast.py` intercepts,
new `stdlib/str.hx`.

### Inc 2 â€” `parse_i32` / `parse_f64` (LOW, ~150 LOC)

Add: `parse_i32(start, len) -> Result<i32, i32>` and
`parse_f64(start, len) -> Result<f64, i32>`. Lower to arena
scan + accumulate. Uses Stage 49 Result<T,E> runtime tag.

### Inc 3 â€” Runtime-path file I/O (MEDIUM, ~200 LOC)

Add: `read_file_to_arena_dyn(path_start, path_len) -> i32`
and `write_file_to_arena_dyn(path_start, path_len,
data_start, n_bytes) -> i32`. Path arg is runtime arena
region, not literal. Requires new IR opcode `FILE_OPEN_DYN`
in `tir.py` + x86_64 backend handler that emits `lea rdi,
[arena_base + path_offset]` instead of RIP-relative LEA.

**Cascading-defect risk:** backend reg-alloc invariants
may need adjustment. Run cascade-verification gate.

### Inc 4 â€” Granular `io.read_file` / `io.write_file` / `io.print` labels (LOW, ~100 LOC)

Sub-label hierarchy in `OP_EFFECTS`. Subsumption: `io`
implies all 3 sub-labels; sub-labels don't imply each other.
Parser change check: does `@effect(io.read_file)` tokenize
with the dot included in the label, or as separate tokens?

### Inc 5 â€” `str_concat` / `str_from_i32` / `str_from_f64` (LOW, ~150 LOC)

Build runtime filenames like `"epoch_" + str_from_i32(epoch)
+ ".bin"`. Lower to __arena_push loops over existing
arithmetic.

### Inc 6 â€” CSV + MNIST stdlib (LOW, Helix source only)

`helixc/stdlib/csv.hx` â€” line iterator scanning for `\n`.
`helixc/stdlib/mnist.hx` â€” IDX format binary blob reader.
Pure Helix source built on Inc 1-5 primitives; no compiler
changes. Cascade-safe.

### Inc 7 â€” Checkpoint serialize/deserialize stdlib (LOW, Helix source only)

`helixc/stdlib/checkpoint.hx` â€” `checkpoint_save(path_start,
