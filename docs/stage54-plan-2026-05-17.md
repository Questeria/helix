# Stage 54 plan — Tier 1 #2: AD across user-defined function calls (broader coverage)

**Date:** 2026-05-17 (opened immediately after STAGE 52 CLOSED at `ba82013`)
**Source:** Tier 1 #2 exploration agent during Stage 52 gate-12 wait
**Estimated complexity:** 3 increments, 1-2 audit gates each

## Stage Goal

ROADMAP Tier 1 #2 ("AD across user-defined function calls")
currently has "In progress" status: forward/reverse AD inline
supported pure helper calls before differentiation, and opaque/
bodyless calls now fail closed instead of producing a zero-
gradient surrogate. **Remaining work is broader chain-rule
registration and richer helper coverage** — Stage 54 ships
those increments.

## Current state (post-Stage 52)

`_diff_call_chain_rule` in `helixc/frontend/autodiff.py` (~line
1169) and `_propagate` in `autodiff_reverse.py` (~line 196)
together cover ~22 single-arg chain-rule arms (transcendentals,
fuzzy logic, identity wrappers). Multi-arg arms exist for
`__powi` and `__bce`.

Three gaps remain:
1. **Multi-arg builtins missing chain-rule arms** — 11 names in
   `AD_KNOWN_PURE_CALLS` (`__min`, `__max`, `__clamp`, +
   `_i32`/`_f64` variants, `__sign`/`__sign_f64`) are pure but
   hit the opaque-call catchall on differentiation.
2. **Forward/reverse asymmetry on unrecognized multi-arg** —
   forward returns zero+warn, reverse raises hard. Same program
   silently produces wrong gradient in forward mode but fails
   loudly in reverse.
3. **`_inline_user_calls` doesn't descend into loop bodies** —
   pure helper calls inside `for`/`while`/`loop` bodies are
   left as-is, never inlined, so any subsequent differentiation
   sees opaque calls. Also: directly-recursive helpers always
   left opaque (no bounded unrolling).

## Increment breakdown

### Inc 1 — Chain-rule arms for __min/__max/__clamp/__sign (LOW complexity)

Add explicit chain-rule dispatch in both modes:

- `__min(a, b)`: `adj_a = adj * indicator(a <= b)`,
  `adj_b = adj * indicator(b < a)`
- `__max(a, b)`: symmetric
- `__clamp(x, lo, hi)`: `adj_x = adj * indicator(lo <= x <= hi)`,
  `adj_lo = adj_hi = 0` (non-differentiable in practice)
- `__sign(x)`: derivative is 0 distributionally; emit
  `FloatLit(0.0)` instead of the opaque-call error
- Apply to all _f64 and _i32 variants

Requires:
- New helper `_chain_rule_min_max_clamp_sign(call, var, span)`
  in `autodiff.py` (forward) and `_propagate_min_max_clamp_sign`
  in `autodiff_reverse.py` (reverse)
- Lift the `if len(call.args) != 1: return None` gate at the
  top of `_diff_call_chain_rule` (or branch on the multi-arg
  cases before it)

Tests:
- 6 new forward tests (one per arm × 2 modes)
- 6 new reverse tests
- 1 parity test for `__clamp`-in-loss

Estimated: ~120 LOC, 1 audit gate.

**Cascading-defect risk:** standard deepcopy discipline (same
hazard as the `__relu` arm audit). Pattern is well-understood
post Stage 36.

### Inc 2 — Forward/reverse asymmetry fix (LOW)

In `_diff_call_chain_rule`, change the `len(call.args) != 1:
return None` fallthrough to emit `_ad_warn()` explicitly. The
`_diff` caller should treat `None` from the chain-rule helper
as "opaque call, warn and return zero" rather than just
fallthrough.

Tests: 1 test asserting `differentiate(opaque_2arg_call, "x")`
emits a warning.

Estimated: ~15 LOC, no audit gate (pure diagnostic).

### Inc 3 — Loop-body descent + bounded recursive unrolling (MEDIUM)

Two sub-parts:

A. Extend `_inline_user_calls`'s `go()` walker to recurse into
   `For`/`While`/`Loop` body subtrees. Currently these are
   returned as-is, leaving any pure-helper calls inside the
   loop unresolved.

B. Add bounded unrolling for directly-recursive helpers. When
   `_inline_user_calls` encounters a function already in
   `visiting`, if the recursive call's argument is a literal-
   reduced subexpression AND the unroll depth is below
   `max_unroll=3`, substitute one more level.

Tests:
- Helper called inside for-loop body (forward + reverse)
- Helper called inside while-loop body (forward + reverse)
- Directly recursive `fn power(x, n)` with literal n=3
  (forward + reverse, verify cubic gradient)
- Negative: mutual recursion left opaque, fails closed

Estimated: ~200 LOC across both autodiff.py and
autodiff_reverse.py. 2 audit gates expected.

**Cascading-defect risk:**
- Loop-body descent: deepcopy discipline (cycle 2 C1 class)
- Bounded unrolling: cache-key signature must include unroll
  depth or Stage C52-AD1-class cache corruption
