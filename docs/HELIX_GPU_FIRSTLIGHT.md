## The kovc-emitted PTX (verbatim)

```ptx
.version 8.0
.target sm_75
.address_size 64

.visible .entry vector_add(.param .b64 param_0, .param .b64 param_1, .param .b64 param_2, .param .b64 param_3)
{
    .reg .pred  %p<256>;
    .reg .b32   %r<256>;
    .reg .b64   %rd<256>;
    .reg .f32   %f<256>;
    .reg .b16   %h<256>;

    mov.u32 %r0, %ctaid.x;
    mov.u32 %r1, %ntid.x;
    mul.lo.s32 %r2, %r0, %r1;
    mov.u32 %r3, %tid.x;
    add.s32 %r4, %r2, %r3;
    ld.param.u64 %rd0, [param_0];
    cvta.to.global.u64 %rd1, %rd0;
    mul.wide.s32 %rd2, %r4, 4;
    add.s64 %rd3, %rd1, %rd2;
    ld.global.f32 %f0, [%rd3];
    ld.param.u64 %rd4, [param_1];
    cvta.to.global.u64 %rd5, %rd4;
    mul.wide.s32 %rd6, %r4, 4;
    add.s64 %rd7, %rd5, %rd6;
    ld.global.f32 %f1, [%rd7];
    add.f32 %f2, %f0, %f1;
    ld.param.u64 %rd8, [param_2];
    cvta.to.global.u64 %rd9, %rd8;
    mul.wide.s32 %rd10, %r4, 4;
    add.s64 %rd11, %rd9, %rd10;
    st.global.f32 [%rd11], %f2;
    ret;
}
```

This is unmistakably kovc's own codegen (distinct from nvcc): hardcoded `.target sm_75` (the box is native sm_86; nvcc would emit sm_86), fixed over-allocated `%r<256>/%rd<256>/%f<256>` register files, and the address computation (`ld.param â†’ cvta.to.global â†’ mul.wide.s32 Â·,4 â†’ add.s64`) re-emitted for **every** array access with no common-subexpression elimination. nvcc emits a `// Generated by NVIDIA NVVM Compiler` banner, minimal registers, and one hoisted index â€” none of which appear here.
